# 步骤 2：逐个挖素材

> 目标：对指定选题，在各平台搜索热门文章，提取核心观点、关键数据、引用
> 输入：步骤 1 产出的 `plan/hot-topics/{YYYY-MM-DD}.md` 中的综合推荐选题
> 产物：`plan/sources/{YYYY-MM-DD}-{选题名}.md`

## 执行流程

### 2.1 确定选题

读取 `plan/hot-topics/{YYYY-MM-DD}.md`，从"综合推荐选题"中选择第一个未处理的选题。
如果需要指定选题，用户可直接告知选题名称。

### 2.2 搜索素材

用 WebSearch 针对该选题在各平台搜索：

**小红书方向：**
- `{选题名} 广州`
- `{选题名} 探店 广州`
- `{选题名} 攻略 广州`

**微博方向：**
- `{选题名} 广州`
- `{选题名} 热门`

**大众点评/本地资讯方向：**
- `{选题名} 广州 推荐`
- `{选题名} 广州 排行`

对每个搜索结果，提取：
- 原文标题和链接（如可获取）
- 核心观点（2-3 条）
- 关键数据或引用
- 亮点句子（可直接引用的精彩表述）

### 2.3 输出素材文件

将收集的信息整理为 `plan/sources/{YYYY-MM-DD}-{选题名}.md`，格式如下：

````markdown
# 素材：{选题名}

> 搜集日期：{YYYY-MM-DD}
> 状态：素材就绪 / 素材不足

## 小红书

### 来源 1：{笔记标题}
- 核心观点：{观点}
- 关键数据：{数据}
- 亮点句子："{引用}"
- 参考链接：{链接}

### 来源 2：{笔记标题}
...

## 微博

### 来源 1：{博文标题}
...

## 大众点评 / 本地资讯

### 来源 1：{文章标题}
...

## 综合提炼

- **共同点**：各来源一致提到的信息
- **差异点**：各来源有分歧或不同角度的信息
- **可用角度**：
  1. {角度 1：适合作为文章主线}
  2. {角度 2：适合作为补充信息}
  3. {角度 3：适合作为争议点/讨论点}

## 待核实信息

- {信息 1} — 需要实地验证
- {信息 2} — 需要确认时效性
````

### 2.4 完成提示

告知用户素材文件已保存。询问：
1. 是否继续处理下一个选题（从综合推荐清单中选下一个）
2. 是否对当前选题进入步骤 3 合成稿
3. 或者结束本次工作
